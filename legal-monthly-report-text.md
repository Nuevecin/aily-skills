# 法务月报生成技能 - 文本版本

## 技能信息
- **技能名称**: legal-monthly-report
- **技能标签**: 法务月报生成
- **技能描述**: 专业法务团队月报生成技能，基于模板结构和月度工作素材自动生成结构化的飞书云文档月报。使用场景：(1) 当用户需要将团队成员（如Davis、Doris、Edwin、Ariel等）的月度工作报告整合为统一格式的团队月报时；(2) 当用户需要基于现有模板（如《综合法务一组工作月报 - 2025年12月》）生成新一期月报时；(3) 当用户需要将多个会议纪要、工作记录等素材内容整合到结构化报告中时。

## 核心功能

### 1. 6维度分类系统
技能严格按照以下6个维度组织重点事项概要：

1. **跨地区项目**
   - 涉及多个地区协同推进的重大项目
   - 典型项目：Project O、Project Stablecoins、法务中心AI能力建设
   - 核心成果提炼要点：交易文件审阅完成、信息共享体系建立、试用方案确定

2. **香港市场**
   - 香港地区专项法务工作
   - 典型项目：6号牌申请、澳门申牌、SFC问询、智能投顾、争议解决案件
   - 核心成果提炼要点：申请回复按时提交、文件已递交、监管认同、调解日期提议

3. **美国市场**
   - 美国地区监管合规与业务支持
   - 典型项目：数据合规项目、AI合规自查、FINRA问询、侵权投诉
   - 核心成果提炼要点：系统字段核对进度、清单完成、信息收集完成、风险可控

4. **加拿大市场**
   - 加拿大地区牌照申请与监管沟通
   - 典型项目：Crypto牌照申请、高管变动、监管问询
   - 核心成果提炼要点：时间线明确、审批中、妥善回复

5. **Web3业务**
   - 各地区Web3/Crypto业务进展
   - 典型项目：集团流动性网络、各地区牌照申请
   - 核心成果提炼要点：架构方案成型、申请材料已递交、补充问题回复准备中

6. **团队管理**
   - 团队内部管理与优化
   - 典型项目：工作流程SOP建立、人员交接、培训支持
   - 核心成果提炼要点：SOP初步建立、交接有序进行、培训体系建立

### 2. 总监视角成果提炼原则
- **拒绝流水账**：不写"修改了邮件"、"参加了会议"等过程描述
- **聚焦里程碑**：将动作转化为结果，如将"和业务开会讨论"转化为"已与业务达成一致，方案定稿"
- **突出决策结果**：强调关键决策点和对业务的影响

### 3. 智能内容合并
- 同一项目多来源合并：如果多位组员提到同一项目，合并为一条完整汇报
- 地区维度智能归类：分析项目涉及地区，归入对应市场维度
- 内容清洗与更新：全部替换为指定月份新进展，严禁保留旧数据

## 工作流程

### 第一阶段：素材分析与准备
1. 读取模板文档，理解结构布局、标题层级、表格格式
2. 分析所有工作素材，提取指定月份的工作进展
3. 按6个维度对事项进行初步分类

### 第二阶段：重点事项概要重构
1. 严格按照6个维度组织重点事项概要
2. 采用总监视角提炼核心成果
3. 执行智能合并，避免内容分散和重复

### 第三阶段：详细内容填充
1. 在模板表格位置创建对应的飞书表格
2. 填入本月素材中的新数据
3. 组织各市场详细内容，保持表格格式与模板一致

### 第四阶段：质量检查与优化
1. 确认所有内容均为指定月份新进展
2. 检查格式与排版规范
3. 验证语言精炼专业，去除AI解释性语言

## 使用示例

### 用户请求示例1：
```
# Role
你是由法务知识库驱动的智能助理，现在请担任综合法务一组的 Leader。

# Context
我已上传了两个类型的文件：
1. **素材文件**（包含 Davis、Doris、Edwin、Ariel 的月报和其他重点工作会议纪要）
2. **模板文件**（综合法务一组工作月报 - 2025年12月）

# Task
请根据素材内容，模仿模板的结构，**直接生成一份新的飞书文档**，标题为《综合法务一组工作月报 - 2026年1月》。
```

### 用户请求示例2：
```
请根据素材内容，模仿模板的结构，**直接生成一份新的飞书文档**，标题为《综合法务一组工作月报 - 2026年1月》。

# Requirements
1. **第一部分"重点事项概要"的强制格式（关键）：**
* 请忽略模板中该部分的旧分类，**必须**按照以下 6 个维度对本月素材进行归类总结。
* **分类维度：** 跨地区项目、香港市场、美国市场、加拿大市场、Web3业务、团队管理。
* **内容要求：** 请分析素材中各项目的所属地区，将其归入对应类别。
```

## 输出格式

最终输出为飞书云文档（.feishu.md格式），包含：
1. 完整的文档结构，复刻模板布局
2. 飞书表格，填入最新数据
3. 专业排版，符合法务报告标准
4. 总监视角的核心成果提炼

## 关键检查清单

### 必须满足：
1. 重点事项概要严格按6维度组织
2. 所有内容均为指定月份新进展
3. 总监视角成果提炼到位
4. 表格格式与模板一致
5. 文档结构完整，排版专业

### 建议满足：
1. 语言精炼，无冗余描述
2. 智能合并逻辑合理
3. 风险与机会平衡描述
4. 交付时间在预估范围内
5. 用户满意度高

1. **创建技能目录结构**：
   ```
   legal-monthly-report/
   ├── SKILL.md
   └── references/
       ├── classification_dimensions.md
       ├── workflow_checklist.md
       └── example_output_structure.md
   ```

   ```bash
   zip -r legal-monthly-report.skill legal-monthly-report/
   ```

3. **重命名扩展名**：
   ```bash
   mv legal-monthly-report.zip legal-monthly-report.skill
   ```
